# Anhang

Dieser Bereich erläutert praktische Anwendungsbeispiele von RxJS und Nutzungsmethoden in spezifischen Bereichen.

Das Ziel ist es, das Verständnis von RxJS weiter zu vertiefen, indem spezialisierte Themen und praktische Anwendungsbeispiele bereitgestellt werden, die im Hauptleitfaden nicht behandelt werden.

## Inhalte des Anhangs

| Thema | Überblick | Zielgruppe |
|---------|------|---------|
| [Eingebettete Entwicklung und Reaktive Programmierung](/de/guide/appendix/embedded-reactive-programming) | RxJS-Nutzung in eingebetteten Systemen, Sensorintegration, Ereigniskorrelationserkennung | Embedded-Entwickler, IoT-Ingenieure |
| [Reaktive Methoden jenseits von ReactiveX](/de/guide/appendix/reactive-patterns-beyond-rxjs) | 7 Methoden wie ereignisgesteuert, Zustandsmaschine, Datenfluss-Programmierung | Regelungstechnik-Ingenieure, Embedded-Entwickler |
| [Gesamtübersicht der Reaktiven Architektur](/de/guide/appendix/reactive-architecture-map) | 7-Schicht-Struktur: UI, Kommunikation, Backend, Datenpipeline, IoT, Steuerung, UX | Full-Stack-Ingenieure, Architekten |
| [Reactive Programming Reconsidered](/de/guide/appendix/reactive-programming-reconsidered) | Lücke zwischen Ideal und Realität von RP, Stärken und Grenzen, Bewertung der Anwendungsbereiche | Alle Entwickler, Architekten |
| [RxJS und Reactive Streams Ökosystem](/de/guide/appendix/rxjs-and-reactive-streams-ecosystem) | Gesamtbild des Technologie-Stacks wie RxJS, Reactor, Kafka und Zusammenarbeit von UI bis Datenschicht | Full-Stack-Ingenieure, Backend-Ingenieure |
